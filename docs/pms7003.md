# PMS7003 Sensor

Plantower PMS7003 particulate matter sensor. Measures PM1.0, PM2.5, and PM10 concentrations.

## Output fields

| Field | Unit | Description |
|-------|------|-------------|
| `PMS1` | µg/m³ | PM1.0 — average of last 10 valid frames |
| `PMS25` | µg/m³ | PM2.5 |
| `PMS10` | µg/m³ | PM10 |

## Power cycle behaviour

The PMS7003 is not left running continuously. It is powered on for `onTime` seconds before each read cycle, during which incoming frames (~1 per second) are collected into a 10-frame ring buffer. Once `onTime` elapses, the averaged result is published and the sensor is powered off again.

`onTime` must be at least 30 seconds (enforced). A longer value gives the sensor more warm-up time and produces a larger average.

## Hardware interface

The sensor communicates over UART at 9600 baud. On ESP8266 this is UART0 (GPIO1 TX / GPIO3 RX), configured via `uart_rx` / `uart_tx` in **Hardware Setup**.

Power and sleep control use two optional GPIO pins, configured independently (see below).

## Configuration

### Global hardware config (`hw.json` / Hardware Setup)

| Field | Key | Description |
|-------|-----|-------------|
| 5 V boost enable | `5v_pin` | GPIO driven HIGH to power the sensor. Set to `-1` if the sensor is always powered or controlled by another means. |
| UART RX | `uart_rx` | GPIO receiving PMS7003 data frames. |
| UART TX | `uart_tx` | GPIO reserved for PMS7003 TX (unused by firmware, set as needed by hardware). |
| On-time | `onTime` | Warm-up duration in seconds (≥ 30, default 30). |

### Per-sensor config (`sensorsetup.json` / Sensor Setup)

| Field | Key | Default | Description |
|-------|-----|---------|-------------|
| SET pin | `set_pin` | `-1` | GPIO connected to the PMS7003 SET input. `-1` = not connected; SET floats or is tied high by hardware, and the sensor is controlled entirely via the 5 V rail. |
| SET inverted | `set_inverted` | `true` | Set to `true` when `set_pin` drives SET through an inverting FET (GPIO LOW → SET HIGH = run; GPIO HIGH → SET LOW = sleep). Set to `false` for a direct connection. |

Example `sensorsetup.json` entry with no SET pin (5 V rail control only):

```json
{"type": "pms7003", "enabled": true}
```

Example with SET pin via inverting FET on GPIO12:

```json
{"type": "pms7003", "enabled": true, "set_pin": 12, "set_inverted": true}
```

## Board variants

| Variant | Power control | SET control |
|---------|--------------|-------------|
| 5 V rail only (default) | `5v_pin` in hw config | none — `set_pin = -1` |
| 5 V rail + inverting FET on GPIO12 | `5v_pin` in hw config | `set_pin = 12`, `set_inverted = true` |
| Direct SET on any GPIO | `5v_pin` in hw config | `set_pin = N`, `set_inverted = false` |
